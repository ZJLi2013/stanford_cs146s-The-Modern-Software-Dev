## NotebookLM 风格总结 / 概念图

### 核心概念
```
编程 Agent 模式
├── 人-Agent 协作模式（Human-agent collaboration patterns）
└── Agent 自主度管理（Managing agent autonomy levels）
```

### 要点列表
- **协作模式**：明确人在哪些环节介入（规划、检查点、代码审查、回滚）；何时全权交给 Agent、何时逐步审批。
- **自主度**：从「只补全」到「多步执行+自我纠错」的谱系；按任务风险与可逆性调节自主度，避免过度放权导致难以挽回的错误。
- **Claude Code 实践**：嘉宾分享 Anthropic 如何设计 Claude Code、最佳实践及在复杂代码库中的用法。

### 关键词
`Human-agent collaboration` `Autonomy levels` `Claude Code` `Agent manager` `Best practices`

### 可选 FAQ
- **Q：什么时候该提高 Agent 自主度？**  
  A：任务可逆、有清晰成功标准、且具备良好测试/回滚时；反之（如生产变更、安全相关）应设检查点与人工审批。
- **Q：作业重点？**  
  A：用 Claude Code 完成指定编程任务，体会「当 Agent 经理」的节奏与决策点。

## 讲师 / 对话总结

第 4 周聚焦**人如何与 Coding Agent 协作**：周一讲「如何当好 Agent 经理」——何时委托、何时介入、如何设检查点与验收标准；周五由 **Boris Cherney**（Claude Code 创造者）分享产品设计、最佳实践与 Anthropic 内部如何使用 Claude Code。核心信息：高杠杆来自「人管策略与判断、Agent 管执行」的清晰分工；自主度要随任务类型与风险调节，而不是一味的全自动。
