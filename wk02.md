## NotebookLM 风格总结 / 概念图

### 核心概念
```
编程 Agent 解剖
├── MCP（Model Context Protocol）
├── 工具使用与函数调用（Tool use and function calling）
└── Agent 架构与组件（Agent architecture and components）
```

### 要点列表
- **MCP**：统一协议，让 LLM/Agent 能安全、结构化地调用外部工具与数据源；理解 MCP 是接入 IDE、终端、自定义工具的基础。
- **Tool use / Function calling**：Agent 通过「调用工具」完成读文件、跑命令、查 API 等动作；设计清晰的工具接口能提升可靠性与可维护性。
- **Agent 架构**：从单轮补全到多步规划—工具选择、执行、观察、再规划；组件包括：模型、工具集、上下文管理、控制循环。

### 关键词
`MCP` `Model Context Protocol` `Tool use` `Function calling` `Coding Agent` `Autonomous Doers`

### 可选 FAQ
- **Q：MCP 和普通 API 有什么区别？**  
  A：MCP 为 LLM/Agent 设计，强调上下文、认证与标准化工具描述，便于多客户端复用。
- **Q：作业会写什么？**  
  A：在 AI IDE 中完成「First Steps」，并动手实现一个自定义 MCP Server。

## 讲师 / 对话总结

第 2 周把「会用 LLM」推进到「会造可执行的 Coding Agent」：周一从零搭建一个能调工具的 Agent，讲清架构（模型 + 工具 + 循环）；周五深入 **MCP**，带大家实现自定义 MCP Server，把自家 API 或内部工具暴露给 Agent。核心信息：Agent 的威力来自「工具+上下文+协议」；MCP 是当前生态里连接 IDE、终端与自建工具的重要标准。
