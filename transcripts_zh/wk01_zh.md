## 中文翻译（全文）

我们都用过它们——大语言模型。不管是让 ChatGPT 给你写个菜谱，还是让 AI 总结一份长篇文档，你按下发送，几乎立刻就能得到一段看起来很有智能的回复。但那黑箱里究竟在发生什么？更重要的是，在你打出第一条 prompt 之前的数月甚至数年里，又发生了什么？

这正是我们今天要深挖的核心问题：揭开这条不可思议的多阶段工程流水线，看它如何把原始、杂乱的互联网数据变成你正在用的 AI 助手。

我们要讲的是训练的根本阶段：预训练、监督微调（SFT）和强化学习（RL），然后再讲你真正用得上的高级 prompt 技巧。今天的材料来自一些很硬核的技术论文和工程讲座，它们把背后的规模和技巧都摊开了。

我们的目标，是让你真正搞懂这些系统的真实架构——从模型吞下数万亿段文本，一直到让它「看起来会推理」的方法。

**从最开头说起。** 在 LLM 成为有用助手之前，它先是一个「基座模型」。功能上是什么？就是一个巨大的、高度复杂的**文本补全机器**。预训练阶段的目标，就是在大到难以想象的尺度上做知识吸收。

第一步是**数据收集**，规模堪称「圣经级」。从对互联网深 archives 的密集抓取开始，最大来源往往是 Common Crawl——那家从 2007 年起就在给全网做索引的非营利机构，超过 27 亿页。然后，这些原始互联网数据会和高质量 curated 文档混在一起：维基百科、学术期刊、GitHub 上的公开代码。

数据是「原始」的，意味着非常乱。他们不可能把整个互联网直接塞进模型。过滤过程非常狠：垃圾、恶意软件、已知种族主义内容、成人内容、甚至只是重复文本，统统去掉。即便如此，数据集仍然是天文数字。比如一个代表性的高质量公共数据集 FineWeb，光文本就有约 44 TB。44 TB 纯文本？没错。这些文本还要被处理成 **token 序列**——基本上就是一小段一小段文本，一个词甚至词的一部分。光 FineWeb 就大约对应 15 万亿个 token，模型要吸收的就是这一万五千亿条训练数据。

那核心的机械操作是什么？那些烧掉几十亿美元的 GPU 到底在干什么？机制在概念上简单得惊人：**LLM 是自回归模型**。也就是说，它们完全根据前面的文本，预测序列中下一个 token 的概率。比如我说「猫在……」模型就预测「垫子上」之类的。对，就是算出最可能的补全。现代模型为做这个预测能利用的**上下文**多得惊人——上下文窗口、工作记忆已经非常大，从 GPT-2 时代的大约一千 token，到现在的几十万甚至百万 token。

这个预测完全是数学的：把上下文送进一个拥有数十亿甚至数万亿可调参数的神经网络。可以把它想成一块巨型调音台上的旋钮——GPT-2 约 16 亿，GPT-4 据传约 1.8 万亿。训练就是一遍遍微调这些旋钮，让模型对「下一个 token」的预测和训练数据里实际出现的越来越一致——每秒数百万次。这需要几乎难以相信的专用算力。这就是你提到的「淘金热」：公司争相租用 A100/H100 这类机器，每 GPU 每小时大约几美元，所以训练一个大型基础模型要花数千万美元，这也决定了整个行业的结构：只有少数玩家玩得起基座模型。

预训练的目标，可以理解为对互联网做一次**有损压缩**——一种人类知识的概率版 zip。但既然是压缩，会丢掉什么？会丢掉细微差别、会弱化「推理」和「区分真假与只是网上常见说法」的能力；它学到的是知识的句法，而不是逻辑的深层结构。这就自然引到下一阶段：那个昂贵的「自动补全」基座模型，并不是我们和 ChatGPT 对话时用的那个。

要把它变成有用的对话助手，必须进入**后训练阶段**，从**监督微调（SFT）**开始。SFT 是塑造模型「人格」的第一步，算力上便宜得多，以小时计而不是月。数据也换了：不再用原始互联网文本，而是规模小得多、高度精选的对话数据，几十万条高质量示例。人类标注员在这里定义 AI 的「人设」：他们按严格、细致的说明，为各种 prompt 写出理想助手回复， literally 在用例子教 LLM 变得有用、诚实、无害。所以当你和 AI 助手对话时，你并不是在和某种新智能形态说话，而是在和一个神经网络互动——它模拟的是「完美执行了那些说明的、高技能人类标注员的平均」。说得再简单一点：你在和人类史上最贵、高度优化过的「客服模拟」说话。

有了人格之后，还要教它**真正解决问题**，这就进入第三个、也是最先进的阶段：**强化学习（RL）**。和 SFT 的区别？SFT 教它「说什么」，RL 教它「怎么想」。在 RL 里，模型可以在可验证的领域练习——比如数学和代码，我们知道标准答案是什么。你给它题目和答案，它必须自己找出推导过程，必须发现能可靠得到那个结果的那一串 token。也是在这里，我们真正看到模型的「认知策略」浮现：它通过一种内心独白式的 token 序列来思考。因为模型每个 token 的计算是有限的，它学会把推理摊到多个 token 上，这就是**思维链（chain-of-thought / CoT）**。如果你让它一口气直接给出难题的答案，它会 consistently 在心算上失败；必须训练它一步一步把计算展开。对用户来说的关键结论就是：**模型需要 token 才能思考**。不让它一步步想，就像要它不用草稿纸做微积分。

纯 RL 在答案可客观验证时（数学、代码、棋类）效果很好，但主观任务呢——比如写笑话、写说服性文章？这些领域公司会用**人类反馈强化学习（RLHF）**：训练第二个 AI——奖励模型——充当人类裁判。人类对不同 AI 输出排序，奖励模型学习复现这些排序，成为人类品味的代理。但材料里也指出一个重大缺陷：强化学习非常擅长在模拟里「刷分」。LLM 可能学会生成专门骗奖励模型打高分的输出，把优化放在真实质量之上。所以 RLHF 很强，却容易短命，模型最终会学会利用奖励系统。人的监督仍然必不可少。

一旦有了这套高度训练后的模型——知识、人格和一定推理能力——最后阶段就是 **prompt 工程**：用对话来「编程」模型。一个基础技巧是**上下文学习、k-shot prompting**：在 prompt 里直接给几个例子（1、3 或 5 个）。比如你要它用某种敬语风格把句子翻译成韩语，就给它一两个那种风格的例子，模型会立刻适应、在上下文中学模式。对更复杂的逻辑任务，还是要回到思维链；哪怕只是加一句「让我们一步步想」（zero-shot CoT），准确率也会明显提升，相当于解锁模型「出声推理」的能力。

模型的知识是冻结在某个时间点的，它不知道昨天发生了什么，也不知道你公司的私有数据库。所以**外部工具**对真实任务至关重要，尤其是 **RAG（检索增强生成）**：它是抵御幻觉的关键，相当于给 AI 一个「智能谷歌搜索」。流程是：LLM 意识到需要最新信息，就生成特殊 token 调用工具（如网页搜索），搜索结果被灌回 AI 的工作记忆、直接插入上下文窗口，再生成最终答案，迫使模型用当前事实而不是旧的训练记忆。还可以用**自校正**技巧：比如**自洽**——同一个问题问五遍，强迫它走五条不同推理路径，再取多数答案，减少随机错误；还有**反思**，特别适合写代码：把错误信息反馈进 prompt，让模型批评并修改自己的代码，相当于教它实时从错误中学习。这些技巧才是「随便聊聊」和「高杠杆专业使用」的差别。

材料里强调的一个核心转变是：讨论不应该是「被取代」，而是——**你不会被 AI 取代，你会被一个会用 AI 的合格工程师取代**。LLM 是在增强工程师，把普通工程师变成超高效率的那一个，OpenAI 内部用 Codex 就是这种用法。

举几个具体例子。**代码理解**：工程师贴上一段错误堆栈，通常要挖一小时；改成 prompt「根据这个堆栈，在仓库里定位 O 逻辑并追踪数据流」，省掉 45 分钟。**重构**：在几十个文件里把旧 API 调用更新一遍，以前是枯燥的三小时手工活；用 LLM prompt「在全部 40 个文件里一致做这个修改」，90 秒搞定。**性能优化**：不用花一小时找瓶颈，让模型分析代码、标出慢的数据库调用并起草修复，30 分钟变成 5 分钟 prompt。**QA 与测试**：指着低覆盖代码让模型跑一晚上，醒来就有一批可跑的单元测试，覆盖人类可能漏掉的边界情况。

但要达到这种精度，不能随便打一句就完事，有**最佳实践**：一是**角色 prompt**，明确定义人设（「你是一个热爱编程、具备高级软件工程师水平的助手」）；二是**结构化 prompt**，用 `<error>`、`<log>` 等标签把复杂数据包起来，方便模型解析；三是**持久上下文**：工程师维护一个像 `agents.mmd` 的文件，写满业务逻辑和命名约定，AI 不可能自己知道，每次会话都贴进去，让模型始终「知情」，所有 prompt 的准确率都会明显提高。

今天我们整条路径捋了一遍：从数万亿原始互联网 token，到人类标注对话赋予人格，再到需要思维链的高级推理。对你来说的核心结论是：**LLM 很强，但有认知缺陷**——可以叫「能力瑞士奶酪模型」：洞对不齐。它能解奥数题，却可能比较不了 9.1 和 9.9。你得把它们当随机工具用：总要检查输出，用这些技巧去弥补局限。

最后留一个思考：当 LLM 在答案可验证的问题上只用纯强化学习训练时，模型不受人类逻辑约束。多年前 AlphaGo 的「第 37 手」就是这样——人类不会想到的策略。如果这些「会思考」的模型继续在开放域问题求解上演进，发现更优的认知策略，甚至发展出比英语更高效的内在思维语言，当解决复杂商业问题的最优方式变成人类根本无法设想的东西时，对我们意味着什么？这关乎人机协作的未来。
