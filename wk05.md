## NotebookLM 风格总结 / 概念图

### 核心概念
```
现代终端
├── 终端自动化与脚本（Terminal automation and scripting）
└── AI 增强型命令行（AI-enhanced command line interfaces）
```

### 要点列表
- **终端自动化**：把重复的 shell 流程脚本化、参数化；与 Agent 结合时可让 Agent 生成或执行安全边界的命令序列。
- **AI 增强 CLI**：Warp 等现代终端将 AI 集成进命令行——自然语言描述意图、自动补全长命令、解释历史命令与错误信息，缩短「想做什么」与「敲对命令」之间的距离。
- **产品视角**：周一从产品角度讲「如何做出一款突围的 AI 开发者产品」；周五由 Warp CEO 分享 Warp 如何用 Warp 构建 Warp，以及 Warp 与 Claude Code 等工具的定位差异。

### 关键词
`Terminal` `Warp` `CLI` `Automation` `AI-enhanced` `Developer tool`

### 可选 FAQ
- **Q：Warp 和 Claude Code 的区别？**  
  A：Warp 侧重终端内体验（命令、工作流、团队协作）；Claude Code 侧重全栈编码与项目级任务；二者可配合使用。
- **Q：作业做什么？**  
  A：用 Warp 进行 Agentic 开发实践，体会 AI 增强终端在真实工作流中的用法。

## 讲师 / 对话总结

第 5 周把场景从 IDE 扩展到**终端**：周一讲打造 AI 开发者产品的思路（谁用、解决什么痛点、如何与现有工具链共存）；周五 **Zach Lloyd**（Warp CEO）分享 Warp 的产品哲学、如何用 Warp 开发 Warp，以及 AI 增强命令行在提升开发者效率中的角色。核心信息：终端仍是许多关键操作的发生地；把 AI 嵌进终端能减少上下文切换、让「想法→命令→结果」更短路径。
